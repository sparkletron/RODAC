The structure

of the architecture of each system is consistent. All architectures are located in the arch folder in the root of RODAC. Each one has its own makefile.mk with the name SYSTEM.mk. Where SYSTEM is the target. For instance coleco.mk is for the colecovision. The base header (base.h) file is for all of the architecture targets and holds defines and functions that are used for all. In the architecture folder named after the target, Colecovision the name would be coleco, contain the header file defines.h and the source files. The defines.h contains the port locations and memory map locations that are used by drivers. The base.c file is the actual functions definitions of the declared functions in base.h. The makefile is a universal build for the target system library. To create a new architecture i t is recommended to copy an existing system and rename it. 
$ cp -r coleco example
$ cp coleco.mk example.mk
The above commands will copy and rename the folder and all its contents to example, and copy the makefile to a new file named example.mk.
